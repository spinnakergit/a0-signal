Installation Mode
signal-cli runs natively inside the Agent Zero container. Simplest setup — no separate Docker container needed. Run python initialize.py --integrated to install.
Uses a separate signal-cli-rest-api Docker container (bbernhard). See docs/SETUP_SIGNAL_API.md for setup.
Signal API Connection
The local signal-cli daemon runs on 127.0.0.1:8080. Usually no configuration needed.
Requires a running signal-cli-rest-api instance. Keep the API on a private Docker network — never expose to the internet.
Leave blank for default (127.0.0.1:8080).
Internal Docker URL of the signal-cli-rest-api container (e.g. http://signal-api:8080).
Only needed if using a secured proxy (e.g. secured-signal-api).

Elevated Mode (2FA)
When enabled, authenticated users can unlock full Agent Zero access from Signal using !auth <key>. Default mode is restricted (chat only, no tools).
Share this key securely (in person or via another E2E encrypted channel). Users send !auth <key> via Signal to elevate.
How long elevated sessions last before requiring re-authentication.
Security warning: "Never expire" means an elevated session stays active until the bridge restarts or the user sends !deauth. If the auth key is compromised, the attacker retains full agent access indefinitely.
